# 라벨 .txt 예시 (복사용 템플릿)

YOLO 라벨 `.txt`는 **숫자만** 허용(주석 불가)이라, 형식 설명은 여기에 둔다.
전체 라벨링 규칙: [`../LABELING_GUIDE.md`](../LABELING_GUIDE.md).

> 아래 좌표는 **형식 설명용 예시값**(특정 도면 아님). 실제 라벨은 라벨링 도구로 생성.
> 좌표는 전부 **이미지 크기로 0~1 정규화**. 파일명: `labels/<split>/<이미지stem>.txt`.

## 1. 레이아웃 det — `layout_example.txt`
형식: `class cx cy w h` (중심좌표·폭·높이). 클래스: `0=title_block 1=view 2=note`.

| 줄 | 의미 |
|---|---|
| `0 0.7900 0.8700 0.4200 0.2600` | **title_block** — 우하단 표제란(가로형 도면 가정) |
| `1 0.3000 0.4000 0.4500 0.5500` | **view** — 정면도(뷰 + 그 치수 포함되게 넉넉히) |
| `1 0.7000 0.4200 0.2500 0.5000` | **view** — 단면도 |
| `2 0.6200 0.6800 0.2000 0.0600` | **note** — 주기 텍스트 블록 |

- 세로형 도면이면 title_block은 보통 하단 띠(예: `0 0.5000 0.9100 1.0000 0.1800`).
- 객체 없는 이미지는 **빈 .txt**(배경 학습).

## 2. 주석 OBB — `obb_example.txt`
형식: `class x1 y1 x2 y2 x3 y3 x4 y4` (4코너, 시계방향 권장). 클래스: `0=measure 1=gdt 2=radii`.

| 줄 | 의미 |
|---|---|
| `0 0.310 0.100 ... 0.310 0.140` | **measure** — 치수+공차(수평 박스 예) |
| `1 0.300 0.150 ... 0.300 0.195` | **gdt** — 기하공차 프레임 |
| `2 0.120 0.300 ...`(살짝 기울어짐) | **radii** — 반경(회전 박스 예: 4코너가 평행사변형) |

- 기울어진 치수/반경은 **4코너를 글자 방향으로 회전**시켜 입력(수평 박스로 만들지 말 것).
- 모따기(`1×45°`)는 `measure(0)`로, 반경(`R..`)은 `radii(2)`로.

## 3. 검증 팁
- 클래스 인덱스가 `drawing_layout.yaml`/`drawing_obb.yaml`의 `names` 순서와 일치하는지.
- 좌표가 0~1 범위인지(픽셀값 X).
- 라벨링 도구의 export 포맷: 레이아웃=「YOLO」, 주석=「YOLO-OBB」.
