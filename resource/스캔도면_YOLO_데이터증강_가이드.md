## 1. 핵심 원리: 증강의 두 가지 목적

증강을 무작정 섞으면 효과를 분석할 수 없습니다. 도면 검출에서 증강은 목적이 둘로 나뉩니다.

| 목적 | 무엇을 하나 | 대표 기법 |
|------|------------|-----------|
| **① 일반화 (variation)** | 위치·크기·맥락 변화에 둔감해지게 | 회전, 스케일, copy-paste, mosaic |
| **② 도메인 정합 (domain matching)** | 학습 데이터를 **실제 스캔본 분포**에 맞춤 | 노이즈, 압축, 번짐, 기울임 |

> **스캔 PDF에서는 ② 도메인 정합이 핵심입니다.**
> 깨끗한 도면으로만 학습하면 실제 스캔본에서 성능이 무너지기 때문입니다.

## 2. 전체 흐름 한눈에 보기

```
[스캔 PDF]
   │
   ▼
① 전처리 (최소화 — 학습·추론 공통 적용)
   ├─ DPI/해상도 정규화        ✅ 유지 (작은 기호 보존)
   ├─ Deskew (기울기 보정)     ⚠️ 큰 기울임만 / 작으면 증강으로
   ├─ 회색조 + CLAHE           ✅ 이진화 대신 권장
   └─ 강한 이진화·denoise      ❌ 생략 권장
   │  (※ 항목별 판단 근거는 §3 참고)
   ▼
② 데이터 증강 (학습 셋에만 적용)
   ├─ 기하학적 (약하게)
   ├─ 스캔 열화 (강하게) ★핵심
   ├─ 명암
   └─ 객체 단위 (copy-paste, mosaic)
   │
   ▼
③ 학습 (YOLOv11)
```

> **중요**: 전처리(①)는 **학습·추론·증강 모두에 일관되게** 적용해야 합니다.
> 추론 때만 전처리를 빼먹으면 증강 효과가 무너집니다.

## 3. 전처리, 어디까지 필요한가 (항목별 판단)

흐름도의 전처리 4단계가 **모두 필수는 아닙니다.** 딥러닝 검출 모델은 고전 OCR과 달라서, 변동을 "전처리로 없앨지 / 증강으로 견디게 할지"를 항목마다 따로 판단해야 합니다. 특히 **이진화는 잘못 쓰면 성능을 떨어뜨립니다.**

### 3-1. 항목별 필요성

| 단계 | 정말 필수인가 | 판단 |
|------|--------------|------|
| **DPI/해상도 정규화** | 🟢 거의 필수 | 작은 기호의 정보량을 좌우 → 유지 |
| **Deskew** | 🟡 조건부 | 작은 기울임은 회전 증강으로 대체 가능 |
| **이진화** | 🔴 불필요 (종종 해로움) | 보통 빼고 회색조 사용 |
| **노이즈 제거** | 🟡 약하게/생략 | 열화 증강과 모순될 수 있음 |

### 3-2. 왜 그런가

**DPI/해상도 정규화 — 유지.** 진짜 목적은 "DPI 맞추기"가 아니라 **작은 GD&T·거칠기 기호가 충분한 픽셀 수를 갖게 하는 것**입니다. YOLO는 입력을 고정 크기로 resize하므로, 도면 크기가 제각각(A4 vs A1)일 때 큰 도면의 작은 기호가 소멸하지 않도록 충분한 해상도 + 타일링(SAHI)이 중요합니다.

**Deskew — 조건부.** 회전 증강(±몇 도)을 충분히 넣으면 모델이 작은 기울임에 강건해져 별도 deskew가 덜 필요합니다. 단, ① 기울임이 증강 범위보다 크거나 ② axis-aligned bbox를 정확히 맞춰야 할 때(기울어진 내용은 박스가 헐거워짐)는 유지하세요.

**이진화 — 보통 제거. ⚠️** 이진화는 고전 CV/OCR 시대의 단계로, 공격적으로 적용하면 **얇은 선과 작은 기호를 지워버립니다** — 정확히 검출하려는 대상을. CNN은 회색조를 잘 처리하며 정보를 더 많이 보존해 검출에 더 유리한 경우가 많습니다. → **이진화 대신 회색조 + 가벼운 대비 정규화(CLAHE)** 권장.

**노이즈 제거 — 약하게/생략.** 증강 단계에서 노이즈를 일부러 추가해 스캔 도메인에 맞추는데, 전처리에서 제거하고 다시 추가하는 건 모순입니다. 강한 denoise는 디테일을 깎으니 가볍게만 하거나 모델이 증강으로 강건성을 학습하게 둡니다.

### 3-3. 결정 원칙 — 정규화 vs 증강 트레이드오프

모든 전처리는 두 선택지의 저울질입니다.

- **전처리로 정규화** → 입력 분포가 좁아져 학습이 쉬워짐. 단, **추론 때도 똑같이 적용해야 하고** 정보 손실 위험(특히 이진화).
- **증강으로 견디게** → 정보 보존, 모델이 변동을 흡수. 단, 데이터·모델 용량이 더 필요.

> **핵심**: 전처리 단계를 하나 추가할 때마다, 학습과 추론 사이에서 어긋날 수 있는 지점이 하나 늘어납니다. **꼭 필요한 것만 남기는 것**이 유지보수와 성능 양쪽에 유리합니다.

### 3-4. 확인 방법 — ablation

위 권장은 일반적 기본값이며 데이터마다 정답이 다릅니다. 같은 데이터로 ① 전처리 전부 / ② 회색조+해상도 정규화만 / ③ 전처리 거의 없음 세 조건의 검증 성능을 비교하세요. **특히 이진화 유무를 따로 비교**해 보면 보통 뺀 쪽이 작은 기호에서 더 좋게 나옵니다.

## 4. 적용 방식과 라벨(bbox) 동기화

### 3-1. 온라인 vs 오프라인

| 방식 | 설명 | 적합한 경우 |
|------|------|------------|
| **온라인 증강** (권장) | 학습 중 매 epoch마다 실시간 변형 | 가벼운 변형 전반 |
| **오프라인 증강** | 미리 변형본을 파일로 생성 | copy-paste 합성 등 무겁거나 검수가 필요한 작업 |

→ **두 방식을 섞는 것이 최선**: 무거운 합성은 오프라인, 가벼운 변형은 온라인.

### 3-2. 가장 중요한 원칙 — bbox 동기화

기하학적 변형을 하면 **bounding box 좌표도 똑같이 변형**돼야 합니다.

- 직접 구현하면 실수가 잦으므로 **Albumentations** 같은 bbox-aware 라이브러리 사용.
- 회전·크롭 후 box가 이미지 밖으로 나가면 자동 제거/클리핑되도록 `min_visibility` 설정.
- **학습 전 반드시 몇 장을 시각화**해서 box가 정확히 따라오는지 눈으로 확인.

## 5. 증강 기법 4종 분류

### 4-1. 기하학적 증강 — **"약하게"**

도면은 방향과 텍스트가 의미를 가지므로 사진처럼 막 변형하면 안 됩니다.

**✅ 안전 (권장 범위)**

| 기법 | 범위 | 이유 |
|------|------|------|
| 약한 회전 | ±2~5° | 스캔 기울어짐 모사 |
| 약한 스케일/줌 | 0.8~1.2배 | 크기 변화 일반화 |
| 약한 평행이동 | ±5% | 위치 변화 일반화 |
| 원근·탄성 왜곡 | 약하게 | 스캔 굴곡 모사 |
| 랜덤 크롭/타일링 | — | **소형 기호 비율↑ (매우 효과적)** |

**❌ 금지**

| 기법 | 이유 |
|------|------|
| 좌우/상하 반전 | GD&T 기호 방향성·텍스트 손상 |
| 90°/큰 각도 회전 | 의미 파괴 |
| 강한 전단(shear) | 기호 형태 왜곡 |

---

### 4-2. 스캔 열화 증강 — **"강하게" (스캔 PDF의 핵심)**

실제 스캔 결함을 코드로 재현한다고 생각하세요.

| 기법 | 모사 대상 |
|------|-----------|
| 가우시안/모션 블러 (약하게) | 스캔 초점 흐림 |
| 가우시안 + salt-and-pepper 노이즈 | 스캔 잡티 |
| **JPEG 압축 (quality 30~70)** | **스캔 PDF 강압축 — 실제 갭의 큰 부분** |
| **Dilation / Erosion (morphological)** | 잉크 번짐 / 선 끊김 |
| 종이 질감·얼룩·접힘·그림자 합성 | 스캔 배경 결함 |
| 다운샘플 후 업샘플 | 저해상도 스캔 (작은 기호 뭉개짐) |

> **핵심 팁**: 강도는 **실제 스캔본 품질 분포를 보고 맞추세요.**
> 실제보다 과하게 망가뜨리면 도메인이 오히려 멀어집니다.

---

### 4-3. 명암 증강

흑백·회색조 도면이 많아 **색 변형은 거의 무의미**, 명암만 유효합니다.

| 기법 | 용도 |
|------|------|
| 밝기/대비/감마 (약하게) | 스캔 노출 차이 |
| CLAHE (적응형 히스토그램 평활화) | 흐린 스캔 대비 개선 |
| ~~색조(hue)·채도(saturation)~~ | 흑백 도면에선 생략 |

---

### 4-4. 객체 단위 증강 — **소형·희소 기호 해결의 열쇠**

| 기법 | 설명 | 주의점 |
|------|------|--------|
| **Copy-paste (실제 스캔 패치)** | 실제 도면에서 잘라낸 GD&T·거칠기·치수 패치를 빈 View 영역에 붙이고 bbox 새로 생성 | 경계는 살짝 블렌딩. 희소 클래스를 더 많이 붙여 불균형도 완화 |
| **Mosaic** | 4장을 한 장으로 합쳐 다양한 맥락·스케일 학습 | **밀집 소형 객체에선 객체가 더 작아져 해로울 수 있음** → 약하게 쓰거나 후반 epoch엔 `close_mosaic`으로 끄기 |
| **Cutout / Random Erasing** | 일부를 가려 occlusion에 강건 | 작은 기호를 통째로 지우지 않게 패치 크기 제한 |

> **왜 "실제 스캔 패치"인가?**
> 스캔본에서는 합성 기호와 실제 스캔 기호의 질감 차이가 큽니다.
> 배경(진짜 스캔 종이) + 객체(진짜 스캔 기호) 둘 다 실제여야 도메인 갭이 최소화됩니다.

## 6. 실전 Albumentations 파이프라인

스캔 도면 검출에 맞춘 온라인 증강 예시 (bbox 동기화 포함):

```python
import albumentations as A

transform = A.Compose([
    # --- ① 기하학적 (약하게) ---
    A.Affine(rotate=(-4, 4), scale=(0.85, 1.15),
             translate_percent=(-0.05, 0.05), p=0.7),
    A.RandomResizedCrop(size=(1024, 1024), scale=(0.6, 1.0), p=0.5),
    A.Perspective(scale=(0.02, 0.05), p=0.2),   # 스캔 굴곡

    # --- ② 스캔 열화 (핵심) ---
    A.OneOf([
        A.GaussianBlur(blur_limit=(3, 5)),
        A.MotionBlur(blur_limit=5),
    ], p=0.4),
    A.GaussNoise(var_limit=(10, 50), p=0.4),
    A.ImageCompression(quality_lower=30, quality_upper=70, p=0.5),  # 스캔 압축
    A.OneOf([
        A.Morphological(operation="dilation", p=1.0),  # 잉크 번짐
        A.Morphological(operation="erosion",  p=1.0),  # 선 끊김
    ], p=0.3),
    A.Downscale(scale_min=0.5, scale_max=0.9, p=0.2),  # 저해상도 스캔

    # --- ③ 명암 ---
    A.RandomBrightnessContrast(brightness_limit=0.2,
                               contrast_limit=0.2, p=0.4),
    A.CLAHE(clip_limit=2.0, p=0.2),
],
bbox_params=A.BboxParams(
    format="yolo",          # YOLO 라벨 포맷
    min_visibility=0.3,     # 30% 미만 보이면 box 제거
    label_fields=["class_labels"],
))
```

> **참고**: `Morphological`은 Albumentations 버전에 따라 이름이 다를 수 있습니다.
> 설치 버전을 확인하고, 없으면 OpenCV로 직접 dilation/erosion을 구현해 넣으면 됩니다.

## 7. 적용 순서와 강도 튜닝

### 6-1. 순서 — 실제 스캔 과정을 따라가기

```
기하학적  →  열화  →  압축
(기울어진 종이 → 스캐너가 흐릿하게 찍고 → PDF로 압축)
```

### 6-2. 강도 튜닝 절차

1. **약한 설정으로 시작** → baseline보다 좋아지는지 확인
2. **한 종류씩 강도를 올려가며** 검증 셋 성능 비교 (ablation)
3. 진단:
   - 학습 손실↓ 인데 검증 성능 정체 → 증강이 **너무 약함**
   - 학습이 불안정 → 증강이 **너무 셈**

## 8. 자주 하는 실수 (체크리스트)

- [ ] **전처리와 증강이 따로 노는가?** → deskew·DPI 정규화는 학습·추론·증강에 일관되게 적용
- [ ] **검증/테스트 셋에 증강을 적용했는가?** → 증강은 **학습 셋에만**. 검증은 실제 분포로 평가
- [ ] **라벨 동기화를 확인했는가?** → 기하 변형 후 bbox를 시각화로 검수
- [ ] **과하게 열화시켰는가?** → 실제 스캔본보다 심하면 도메인이 오히려 멀어짐
- [ ] **Mosaic가 소형 객체를 더 작게 만들었는가?** → 밀집 기호엔 약하게/후반 비활성화

## 9. 요약

스캔 도면 증강의 무게중심:

> ### 약한 기하 변형  +  강한 스캔 열화 모사  +  실제 패치 copy-paste

**우선순위**

| 순위 | 항목 | 효과 |
|------|------|------|
